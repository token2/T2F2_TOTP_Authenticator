Supported keys: the Token2 PIN+ series only. Works with Token2 PIN+ series FIDO2 keys (T2F2-PIN+, in their various form factors — Dual, Type-C, Mini, NFC-Slim, and so on, across Release 2 / 3 / 3.x), which can securely store TOTP secrets on the key. Older or non-PIN+ keys, and first-generation keys (plain T2F2, T2F2-TypeC) which only have HOTP, are not supported for TOTP.

What this is

Token2's PIN+ series keys can emulate the TOTP protocol. Doing a TOTP login the long way — find the profile, copy the code, switch to the login page, paste, submit — is seven steps for something you may do many times a day.

T2 TOTP Authenticator collapses that into one keystroke. Its headline feature is Auto-OTP: press a global hotkey and the current one-time code for your tagged profile is typed straight into whatever field has focus (optionally followed by Enter). If the key is plugged in and the app is running, logging in is a single hotkey press.

Auto-OTP — the idea

Instead of the seven-step copy-and-paste dance, press a hotkey and the code is sent to the focused input.

CtrlAltABCFNQSVXZ
For technical reasons the shortcut is limited to left Ctrl + Alt + a letter from a fixed set chosen not to clash with common shortcuts: A, B, C, F, N, Q, S, V, X, Z.

A key can hold up to 50 TOTP profiles, but only one can be used with Auto-OTP. That profile is marked with a special [A] tag appended to its issuer. So a TOTP login becomes three steps — or just one when the key is already plugged in and the app is running.

The interface

The window lists your TOTP profiles with their live codes and a countdown ring. The Auto-OTP profile is flagged with an Auto-OTP badge (the [A] tag). From the header you can pick the transport, add a profile, refresh, open Settings, and Exit.

Download & run

Releasing for Windows. The original project was Windows-only, and this is a Windows-first release. Because it's written in Rust, macOS and Linux builds are possible too — but they are untested, mainly because of OS-specific behaviour around the global hotkey and synthetic keystrokes. See Other platforms.

T2 TOTP Authenticator is a self-contained executable — no installer, no runtime to deploy. Download it, put it somewhere convenient, and run.

  • Admin rights: not required in normal use. If the key has HID USB mode disabled, TOTP data is reachable only over the FIDO channel, which on Windows needs administrator rights — run the app as administrator in that case.

Settings & configuration

Open Settings (the button) to configure Auto-OTP:

  • Shortcut — fixed to Ctrl + Alt + one of the allowed letters.
  • Press Enter after typing the code — sends an Enter keystroke after the code so the login submits automatically.

Settings are saved to a small text file so they persist between runs and can be pre-deployed alongside the executable. On Windows:

%APPDATA%\t2totp\settings.conf
# e.g. C:\Users\<you>\AppData\Roaming\t2totp\settings.conf

It's a simple key = value file:

# which transport to use: auto | hid | nfc
transport = auto

# Auto-OTP hotkey
hotkey_enabled = true
hotkey_key = KeyZ            # KeyA,KeyB,KeyC,KeyF,KeyN,KeyQ,KeyS,KeyV,KeyX,KeyZ
hotkey_append_enter = true   # send Enter after the code

Adding a TOTP profile

You provision a profile from the account's TOTP secret. Below is the Office 365 / Microsoft Entra (Azure) MFA flow as an example.

These are full-featured FIDO2 keys — for Microsoft accounts you can use the more secure passwordless method instead of TOTP MFA, which we recommend whenever it's an option. TOTP MFA remains useful where passwordless isn't available.

Get the TOTP secret

Sign in and open your security info.
Choose Add method → Authenticator app.
When prompted for Microsoft Authenticator, click I want to use a different authenticator app (it uses a protocol hardware tokens can't accept).
Click Next until the QR appears, then Can't scan image? to reveal the secret key. Keep the page open.

Add the secret to your key

In the app, click Add to open the form.
Fill in Issuer (e.g. O365), Account (your username), and paste the Base32 secret. Adjust SHA-256 / digits / period / touch only if the provider needs non-defaults (Microsoft uses the defaults).
Click Add — the profile appears with its live code.
To use this profile with Auto-OTP, tick Auto-OTP [A] (or append the [A] tag to the issuer). This marks the single Auto-OTP profile.
Scan QR from the screen. If built with the optional qr feature, the Add form has a Scan QR from screen button: display the provider's QR code, click it, and issuer, account, secret, algorithm, digits, and period are filled automatically. Capture and decoding are entirely local — nothing is uploaded. No TOTP QR on screen, or no secret in it → a clear message and nothing changes.

Verify

Back on the Microsoft page, click Next and enter the current code. Good first try of Auto-OTP: if you tagged the profile [A], focus the code field and press your hotkey instead of typing the six digits.

FAQ

Can I turn Auto-OTP on for an existing profile?

No — it can only be set when the profile is created; for security the device doesn't allow changing a TOTP slot afterwards. Re-add the profile with the [A] tag if you need it.

Can I have more than one Auto-OTP profile?

The [A] tag is just text, so technically yes — but only one is used; if several are tagged, the list is sorted and the last one wins. Keep it to a single [A] profile.

Why might the app need administrator rights?

Only if the key has HID USB mode disabled — then TOTP data is reachable only over the FIDO channel, which needs admin rights on Windows. With HID enabled (the default), no elevation is needed.

Where are my profiles stored?

On the key itself — never on the PC. The app only reads codes and writes new profiles to the device.

Is the Auto-OTP Enter reliable on rapid presses?

Yes. On each press the app releases any held modifiers, types the digits, then sends a clean Enter — so every press submits correctly.

Other platforms

The app is written in Rust and released for Windows, matching the original project. macOS and Linux are buildable from the same source but are untested. The parts most likely to need platform-specific work are the global hotkey and synthetic keystrokes.

Command-line tool (t2totp)

The project also ships a small command-line tool, t2totp, for scripting and headless use — listing codes, adding/removing profiles, and inspecting the key. It uses the same transports (auto-detected, PC/SC first).

USAGE:
    t2totp [GLOBAL OPTS] <COMMAND> [ARGS]

GLOBAL OPTS:
    --transport <auto|hid|nfc>   Interface to use (default: auto)
    --reader <name>              Pin a specific PC/SC reader (serial-confirmed)
    --debug                      Trace device I/O on stderr

COMMANDS:
    info                         Show the connected key (serial, transport, TOTP support)
    readers                      List PC/SC readers and flag Token2 FIDO keys (by serial)
    list                         List stored profiles with live TOTP codes
    code <issuer> <account>       Print the current code for one profile
    add  <issuer> <account>       Add a TOTP profile (secret via stdin or $T2TOTP_SECRET)
                                   --auto            append the [A] Auto-OTP tag
                                   --sha256          use SHA-256 (default SHA-1)
                                   --period <sec>    TOTP step (default 30)
                                   --digits <4..10>  code length (default 6)
                                   --touch           require a button press to emit the code
    delete <issuer> <account>     Delete a profile
    erase --yes                  Erase ALL profiles on the key
The secret is never passed on the command line. add reads the Base32 secret from stdin or the T2TOTP_SECRET environment variable, so it never lands in shell history or the process list.

Examples

# identify the key, list codes
t2totp info
t2totp list

# print one code (handy in scripts)
t2totp code O365 alice@example.com

# add a profile, piping the secret on stdin
printf '%s' "JBSWY3DPEHPK3PXP" | t2totp add O365 alice@example.com

# add an Auto-OTP-tagged profile, SHA-256, 8 digits
printf '%s' "JBSWY3DPEHPK3PXP" | t2totp add O365 alice@example.com --auto --sha256 --digits 8

# or pass the secret via the environment
T2TOTP_SECRET="JBSWY3DPEHPK3PXP" t2totp add Acme bob

# force NFC, or pin a specific reader
t2totp --transport nfc list
t2totp --reader "ACS ACR1252 1S CL Reader" list

# delete one profile, or wipe the key
t2totp delete O365 alice@example.com
t2totp erase --yes
